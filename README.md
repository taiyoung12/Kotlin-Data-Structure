# Kotlin-Data-Structure
TDD 방식으로 자료구조를 구현합니다. 


### 1. 연결리스트   
### 기능 리스트  
```angular2html
1. 연결리스트는 노드들의 집합이다.
    - head(첫 노드) 와 tail(끝 노드) 노드를 가진다.
    - 노드는 데이터와 다음 노드를 가리키는 참조로 구성된다.
2. 연결리스트는 데이터를 저장할 수 있다.
3. 연결리스트는 데이터를 추가할 수 있다.
    - 임의의 위치에 데이터를 추가할 수 있다 
    - head, tail 에 데이터를 추가할 수 있다.
4. 연결리스트는 데이터를 삭제할 수 있다.
    - 임의의 위치에 데이터를 삭제할 수 있다.
    - head, tail 에 데이터를 삭제할 수 있다.
5. 연결리스트는 데이터를 조회할 수 있다.
6. 연결리스트는 데이터를 수정할 수 있다.
7. 연결리스트가 비어 있을때 핸들링을 할 수 있다.
8. 연결리스트의 노드 갯수를 조회할 수 있다.
```

### 2. 스택  
### 기능 리스트 
```angular2html
1. 스택은 후입선출(LIFO, Last In First Out) 방식의 자료구조이다.
2. 스택은 데이터를 push 하여 저장할 수 있다.
    - 새로운 데이터를 스택의 최상위(top)에 추가할 수 있다.
3. 스택에서 데이터를 pop 하여 제거할 수 있다.
    - 스택의 최상위 노드(top)에 있는 데이터를 제거할 수 있다.
4. 스택에서 데이터를 peek 할 수 있다.
    - 스택의 최상위 노드(top) 데이터를 확인할 수 있다.
5. 스택이 비어 있는지 확인할 수 있다.
    - 스택이 비었을 경우 핸들링을 할 수 있다.
6. 스택의 임의 인덱스의 노드를 알 수 있다.
    - 스택의 임의 인덱스의 노드를 조회할 수 있다.
7. 스택의 크기(노드 개수)를 조회할 수 있다.
    - 스택에 쌓인 데이터의 개수를 조회할 수 있다.
8. 스택에서 데이터를 초기화할 수 있다.
    - 스택에 저장된 모든 데이터를 제거하여 빈 상태로 만들 수 있다.
```

### 3. 큐 
### 기능 리스트
```angular2html
1. 큐는 선입선출(FIFO, First In First Out) 방식의 자료구조이다.
2. 큐는 데이터를 enqueue 하여 저장할 수 있다.
    - 새로운 데이터를 큐의 끝(rear)에 추가할 수 있다.
3. 큐에서 데이터를 dequeue 하여 제거할 수 있다.
    - 큐의 앞(front)에 있는 데이터를 제거할 수 있다.
4. 큐에서 데이터를 peek 할 수 있다.
    - 큐의 앞(front)에 있는 데이터를 확인할 수 있다.
5. 큐가 비어 있는지 확인할 수 있다.
    - 큐가 비었을 경우 핸들링을 할 수 있다.
6. 큐의 크기(노드 개수)를 조회할 수 있다.
    - 큐에 쌓인 데이터의 개수를 조회할 수 있다.
7. 큐의 임의 인덱스의 노드를 조회할 수 있다.
    - 큐의 특정 위치의 데이터를 조회할 수 있다.
8. 큐를 초기화할 수 있다.
    - 큐에 저장된 모든 데이터를 제거하여 빈 상태로 만들 수 있다.
```

### 4. 해시 테이블
### 기능 리스트 
```angular2html
1. 키-값 쌍으로 데이터를 저장할 수 있다.
    - 주어진 키에 대해 값을 저장할 수 있어야 한다.
2. 주어진 키로 데이터를 조회할 수 있다.
    - 키를 통해 해시테이블에서 값을 빠르게 찾을 수 있어야 한다.
3. 주어진 키로 데이터를 삭제할 수 있다.
    - 특정 키에 저장된 데이터를 해시테이블에서 제거할 수 있어야 한다.
4. 해시 충돌을 처리할 수 있다.
    - 동일한 해시 값을 갖는 키들이 있을 때, 충돌을 해결할 수 있는 메커니즘을 제공해야 한다.
5. 해시테이블이 비어 있는지 확인할 수 있다.
    - 해시테이블이 비었는지 여부를 확인할 수 있어야 한다.
6. 현재 저장된 데이터의 개수를 조회할 수 있다.
    - 해시테이블에 저장된 키-값 쌍의 개수를 확인할 수 있어야 한다.
7. 주어진 키가 존재하는지 확인할 수 있다.
    - 특정 키가 해시테이블에 존재하는지 확인할 수 있는 기능을 제공해야 한다.
8. 동적으로 용량을 조정할 수 있다.
    - 해시테이블이 특정 용량을 초과하면 자동으로 크기를 조정하여 더 많은 데이터를 저장할 수 있도록 해야 한다.
9. 해시테이블을 초기화할 수 있다.
    - 해시테이블을 완전히 초기화하여 모든 데이터를 삭제하고 빈 상태로 만들 수 있어야 한다.
10. 모든 키와 값을 순회할 수 있다.
    - 해시테이블에 저장된 모든 키와 값을 순회하는 기능을 제공할 수 있다.
```
